Olden days System admin of the company will check for their software
1. up to date of os
2. Application depenecy are up to date . java / pyhton
3. maintence
4. configuration managment

he was doing it manually . so it was creating the issue
Because they have multiple machine , un=buntu , cent os , debion, alpine, windows , etc

To solve this problem  Puppet ,cheif , salt ,Ansible
6-7 years back  Puppet ,cheif was popular

System admin --->  (Puppet ,cheif) --> comvert to python   ---> Actual Infra ( linux , window, etc )
here challenge is --> learning ruby for puppet and checif
                  --> they are not agent less . means they need to go to each machine and install agent on it .
                  --> pupet/cheif has complex coding
To solve this Ansible came
   -> you have to know just YAML /YML
   -> So every one is happy
   -> Ansible Play books
   --> We are also agent   . so you no need to install anythign on actual infra
   --> Control node : Machine where you install Ansible software called control node
       Manage Node : Actual infa . SHOULD HAVE PYTHON
   --> End to End flow
       create yaml file --> run it in Control node --> All manage node can be update the software etc
Ansbile will do only configuration ?
   --> No
   --> Its powerful automation platform
   --> Configuration Management is one of the feature
   --> Ansbile do Provisiong , similar to terraform how it provising EC2 instance etc
   --> Used for Deployment in CI/CD
   --> Also used for Network automation

Ansible vs Shell vs Python :
 So which one you will choose ?

  --> Shell script to install java  # if one of the vm is windows , ubntu , linux then it wont work
  --> Python u can install   #  you need to know python programming , update the python script , system admin has to know python verl well
  --> Ansible you can install  # no programming language just YAML/yml
