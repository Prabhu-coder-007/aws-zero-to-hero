Authetication vs Authorization

Authentication - whether you have access to enter the bank
Authorization - whether you can withdraw cash , deposit cash , check balance etc

In AWS Root user can do anything in AWS.

IAM - Identity and access Managment
    - It will do Autentication and Authorization
    - Root access not shared with any one
    - Devops/AWS Infra Enginner will give mostly read access to developer
    -

User :
   - Creating a new user for new person in the team
   - Allow them to enter to AWS


Policy :
   - What above user can do it ? that is called Policy.  What he can do it in AWS.

Groups :
   - Everytime when new user comes we need to create a user. if that user leave then we need to remove the user.
   - But we can create a Group , groups are like QA, Dev, DB and others
   - Just add new user to group . so that he will get all the access attached to policy in that group

   - Application which is deployed on On-Premies want to connect the service in AWS then we use Roles
   - If two AWS account want to connect then we use Roles

EC2 Deep Dive :

What is EC2 Instance ?
 - Elastic Cloud Computing (EC2)
 - Compute Means you are requesting AWS to provide virtual server
 - If you want to use your laptop multiple people then use hyberviser in ur laptop . then more than 1 person can use ur lapto.
 - Elastic means scalled up/scalled down . thats why its Elastic services

Why should i use EC2 Instance ?
If its on-premius
 - Timely Upgrade
 - Security Issues
 - Server Down
 -
to get raid of above issues(maintence) we will go to cloud . and cost also less (pay as you use) .

Types of EC2 Instance :
  - 5 Differnt types
  - General purpose
  - Compute Optimize --> HIGH compute - machine leraning
  - Memory Optimized --> HIGH memory - big data analystics etc
  - Storage Optimized --> HIGH storage -
  - Accelarated

Our demo puropse we will use Genral purpose EC2 instance .

What is region ?
 - AWS has data center across world . this is called region .
 - You can ask give me EC2 instance in india.Why india ? you client who use ur application are india . like sbi bank.
 - In india we have Data center in hyderabad , mumbai
 - So that latency is low . request/response will flow fast

Inside region we have mutiple avilabilty zones .
 - In mubai we have 2 availabilty zones , one in north mumbai and one in south mumbai . so that even 1 goes down otehr one will be available.

 Question 2 :
 Your organization has a VPC with multiple subnets. You want to restrict outbound internet access for resources in one subnet,
 but allow outbound internet access for resources in another subnet. how would you achive this ?

 Ans : To restrict outbound intenet access for resources in one subnet, we can modify the route table
 associated with that subnet. in the route table , we can remove the default route (0.0.0.0/0) that points to an internet gateway.

 This would prevent resources in that subnet from accessing the inetnet. for the subnet where outbound internet access is
 required ,we can keeo the default route pointing to the internet gateway .

 Question 3:
 You have a VPC with a public subnet and a private subnet. Instance in the private subnet need to accesss the internet for software updates.
 How would you allow intenet access for instance in the private subnets ?

 Ans : To allwo internet access for instance in the private subnet, we can use NAT Gateway or a NAT instance .

 We would place the NAT Gateway/Instance in the public subnet and configure the private subnet route table to send outbound
 taffic to the NAT Gateway/Instance . This way, instance in the private subnet can access the internet through the NAT gATEWAY/iNSTANCE

  Question 4:

  What steps would you table to enable this communication ?

  Ans : By default, instance within same VPC can communicate with each otehr using private IP address.

  To ensure this communication, we need to make sure that the instances are lanuched in the same VPC and are places in the
  same subnet or subnets that are connected through a peering connection or a VPC peering link.

  Additionaly, we should check the security groups associated with the instance to ensure that the necessary inbound and outbound rules
  are configured to allow communciation between them.
